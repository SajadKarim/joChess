Hey Folks;

Naming conventions we should follow;

(Imp: I am adding it here as there is not sharepoint or central portal of post this info.)

  1. Class member variables should always start with 'm' followed with '_' and short 1/2 letters to represent type of the variable.
    For example: 
    m_oPosition (Object of Position)
    m_nAge (Age in Integer)
    m_stName (Name as String)
    m_ptrRule (Pointer to Rule)
    m_mp (for Map)
    m_lst (for List)
  2. Other variables should follow the same style except "m_" in the begining.
  3. Function names follow Pascal casing, whereas variables follow camel cases.
  4. Avoid magic numbers. Use macros instead. (Preferably in a logically centralized location).
  5. Please avoid using 'this' unnecessarily. 
    5.1 "this.name = name" (here it's required to tell complied about the member variable.
    5.2 "this.m_stName = stName" (here it does not make any sense)
