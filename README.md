# joChess

This was an academic project. In the course "Clean Code Development" we were provided a 2-Players Chess game and we were required to mold it to 3-Players Chess game. While doing the transformation, we were required to apply SOLID principles and use the tools for code management and any CI server.

Our core logic relies on XML. We have tried to make the game (to great extent) configurable where the input XML defines board layout, pieces, rules and players. XMLs for 2 and 3 (two variations â€“ hexagon & circle) players are already added to source control and further boards can be defined using the same pattern. (The detailed architecture diagram shall be added under documentation section of the code.)

The base code used in this project is taken from the following location;
https://sourceforge.net/projects/javaopenchess/files/1.0.1/joChess-1.0.1-sources.zip/download

The engine has to be generic, this requires some contract or predefined rules that all the boards must follow to work properly. In the initial phase, XML is considered as means to store board details, and in the later phase, as the logic is written in data layer - and if required, it can be replaced by Database. Engine&rsquo;s core logic to handle different moves is written in a different module (idea is to load it at run time - dynamic library). The rationale for making it library is to allow developer to make custom rules that cannot be handled using XML.</span></p><p class="c35"><span style="overflow: hidden; display: inline-block; margin: 0.00px 0.00px; border: 0.00px solid #000000; transform: rotate(0.00rad) translateZ(0px); -webkit-transform: rotate(0.00rad) translateZ(0px); width: 624.00px; height: 266.67px;"><img alt="" src="images/image2.png" style="width: 624.00px; height: 266.67px; margin-left: 0.00px; margin-top: 0.00px; transform: rotate(0.00rad) translateZ(0px); -webkit-transform: rotate(0.00rad) translateZ(0px);" title=""></span></p><p class="c35"><span class="c2">Fig 1: Superficial view of system main components.</span></p><p class="c0 c6"><span class="c2"></span></p><hr style="page-break-before:always;display:none;"><h2 class="c37 c63" id="h.mkb750nh3o3a"><span class="c4 c31"></span></h2><h2 class="c37" id="h.9l7mfhenri73"><span class="c31 c4">Board as XML</span></h2><p class="c0"><span class="c2">XML is used to feed the engine about board details that include cells, pieces, rules, players, and the related information. In the initial phase, XML follows the following structure:</span></p><p class="c19"><span class="c7 c4">&lt;Board ...&gt;</span></p><p class="c11"><span class="c7 c4">&lt;Positions/&gt;</span></p><p class="c11"><span class="c7 c4">&lt;Rules/</span></p><p class="c11"><span class="c7 c4">&lt;Pieces/&gt;</span></p><p class="c11"><span class="c7 c4">&lt;Players/&gt;</span></p><p class="c19"><span class="c7 c4">&lt;/Board&gt;</span></p><h3 class="c0 c59" id="h.s0xzj39efv1n"><span class="c5 c4"></span></h3><h3 class="c0" id="h.r96kp9gymaok"><span class="c5 c4">Basics</span></h3><p class="c0"><span class="c2">Board basic details are added under root element &ldquo;Board&rdquo;. These details include board background image, width, height, details regarding rule processing library, etc.</span></p><p class="c0"><span class="c2">Example:</span></p><p class="c25"><span>&nbsp;</span><span class="c7 c4">&lt;Board Name=&quot;2PlayerBoard&quot; BoardImage=&quot;chessboard_2players.png&quot; ActiveCellImage=&quot;sel_square.png&quot; MarkedCellImage=&quot;able_square.png&quot; Width=&quot;480&quot; Height=&quot;480&quot; &nbsp;RuleEngine=&quot;DefaultRuleEngine&quot; RuleProcessor=&quot;DefaultRuleProcessor&quot;&gt;</span></p><p class="c25"><span class="c7 c4">&hellip;</span></p><p class="c25"><span class="c7 c4">&lt;/Board&gt;</span></p><p class="c0"><span class="c2">Where &ldquo;2PlayerBoard&rdquo; is the name of the board. BoardImage is the background image of the board. ActiveCellImage is the image to highlight active cell or position user selects (to give cell selection behaviour).
